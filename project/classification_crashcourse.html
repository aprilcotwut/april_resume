PCA¶

For a centered dataset X, you simply find the covariance matrix Σ such that you can apply eigenvalue decomposition to get eigenvectors, with which you can take the first k columns project the dataset to one representing the k dimensions of maximized variance

colors = ['#FF0000', '#FFA500', '#FFFF00', '#008000', '#0000FF', '#800080', '#FF00FF', '#A52A2A', '#008080', '#808080']
def draw_umap(x, y, n_neighbors, min_dist=0.15, metric='euclidean', title=''):
    # mock the user if they didn't infer the constraints
    assert len(n_neighbors) == 4, "Sad tbh"
    
    # create 2x2 grid
    fig, axs = plt.subplots(2, 2, figsize=(10, 8), subplot_kw={'projection': '3d'})

    # create individual plots    
    for ax, neighbor, i in zip(axs.ravel(), n_neighbors, range(4)):
        fit = umap.UMAP(
            n_neighbors=neighbor,
            min_dist=min_dist,
            n_components=N_COMPS,
            metric=metric
        )

        fitter = fit.fit(x, y=y)
        u = fitter.transform(x)
        
        ax.scatter(u[:,0], u[:,1], u[:,2], c=data_colors)
        ax.set_title(f"neighbors: {neighbor}")
        print(f"Fit umap {i+1} of 4")


    plt.title(title, fontsize=18)

def noise(images,prob):
    for i in range(images.shape[0]):
        if random.random() < prob:
            images[i] = random_noise(images[i] / 255, mode="speckle") * 255
            images[i] = random_noise(images[i] / 255, mode="localvar") * 255
            images[i] = np.clip(images[i], 0, 255)
            images[i].astype(int)

        if j >= imgs_to_show:
            break

Neural Networks¶

A neural network is inspired by how neurons firing in the brain leads to decision making in biological systems. It takes in data usually related to some labeled output we wish to predict and attempts to pick up on the complicated patterns within them. The simplest neural network consists of an input layer, a hidden layer, and an output layer. Based on the values of the data coming in, certain nodes of the hidden layers “light up” (have a higher value) depending on whether the value of the node exceeds some value, it then fires the number to the next connecting node (or to an output if it’s a single layer). For classification, your output is usually some "probability" related to which output it considers most likely.

Random Forest¶

Random Forests is an ensembled learning technique which constructs many decision trees, often limited to only some of the features, and allows them to "vote" on the best solution. Additionally, these trees often use bootstrapped samples (a sampled subset with replacement). If bootstrapping is used, one can use out-of-bag samples (which is, the data not used from sampling) for evaluation of the model, without needing to utilize the test/validation set.

Decision trees work by selecting the best splitting feature and value (node) based on a metric (e.g. Gini impurity, entropy). These splits continue until some stopping criterion is met, e.g. all values in the split belong to one class, maximum tree depth is reached, number of samples exceeds minimum number of samples for split.

Gradient Boosted Machine¶

Gradient Boosted Machines (GBMs) are another ensembled learning approach that utilizes decision trees (usually) sequentially. That is, each new tree learns and corrects the errors of the prior. To do this, GBMs use gradient descent optimization. Each interaction n_classes trees are fit on the negative gradient of the loss function. That is, the model is fit to the residuals (or pseudo residuals) rather than the values. Because of this, the final prediction, rather than being a weighted vote like the Random Forest, is a weighted sum of the predictions, where the weight is determined in the training process.
